Final Project: Automated QA Framework (UI & API)

Project Summary: Automated QA Framework

This project implements a unified End-to-End Automation Testing Framework combining Web UI and Rest API validation within a single repository. Built on Java and Gherkin/Cucumber, it demonstrates a commitment to robust, continuous testing practices.

Lihat Kode di GitHub

Core Technologies

  • Programming Language: Java
  • Build & Dependencies: Gradle
  • Testing Framework: Cucumber (Gherkin)
  • Web UI Testing: Selenium WebDriver
  • API Testing: Rest Assured
  • CI/CD: GitHub Actions

Testing Scope

  • Web UI Target: DemoBlaze (demoblaze.com)
  • Web UI Focus: End-to-End validation (Selecting items, Add to Cart, Checkout flow).
  • API Target: DummyAPI.io
  • API Operations: Comprehensive CRUD (Create, Read, Update, Delete) testing.
  • Framework Pattern: Page Object Model (POM) for maintainability.

How to Run Locally

Run API Tests: ./gradlew apiTest

Run Web UI Tests: ./gradlew webTest

Report Location: build/reports/cucumber/html/index.html
